## Чек-лист тестирования

### 1. Функциональное тестирование

#### 1.1. Кнопка "Список задач"

- [ ] Кнопка отображается только при наличии задач (`tasksCount > 0`)
- [ ] Кнопка не отображается, если задач нет
- [ ] При клике на кнопку происходит переход на уровень 2
- [ ] Кнопка стилизована корректно (primary, с иконкой)

**Шаги:**
1. Открыть попап детализации трудозатрат
2. Проверить наличие кнопки "Список задач" (если есть задачи)
3. Кликнуть на кнопку
4. Проверить переход на уровень 2

---

#### 1.2. Навигация между уровнями

- [ ] Переход с уровня 1 на уровень 2 работает
- [ ] Кнопка "Назад" возвращает на уровень 1
- [ ] Плавные переходы между уровнями работают
- [ ] При возврате на уровень 1 состояние сбрасывается

**Шаги:**
1. Открыть попап уровня 1
2. Кликнуть "Список задач"
3. Проверить отображение уровня 2
4. Кликнуть "Назад"
5. Проверить возврат на уровень 1

---

#### 1.3. Отображение карточек задач

- [ ] Карточки задач отображаются с правильными данными
- [ ] Все поля отображаются корректно:
  - [ ] Номер задачи
  - [ ] Название задачи
  - [ ] Трудозатрата
  - [ ] Дата начала
  - [ ] Дедлайн
  - [ ] Дата завершения
- [ ] Визуальное выделение просроченных задач работает
- [ ] Визуальное выделение завершённых задач работает

**Шаги:**
1. Открыть уровень 2 со списком задач
2. Проверить отображение всех полей в карточках
3. Проверить визуальное выделение (цвета границ)

---

#### 1.4. Пагинация

- [ ] Пагинация отображается только если задач > 10
- [ ] Переключение страниц работает корректно
- [ ] Кнопки "Предыдущая"/"Следующая" работают
- [ ] Активная страница выделена визуально
- [ ] При смене страницы загружаются соответствующие задачи

**Шаги:**
1. Открыть уровень 2 с >10 задачами
2. Проверить отображение пагинации
3. Кликнуть на страницу 2
4. Проверить загрузку задач страницы 2
5. Проверить выделение активной страницы

---

### 2. Тестирование данных

#### 2.1. Корректное отображение полей

- [ ] Название задачи отображается корректно
- [ ] Номер задачи отображается корректно
- [ ] Трудозатрата форматируется правильно (часы)
- [ ] Даты форматируются правильно (DD.MM.YYYY)
- [ ] Отсутствующие поля отображаются как "-"

**Шаги:**
1. Открыть уровень 2
2. Проверить форматирование всех полей
3. Проверить отображение отсутствующих полей

---

#### 2.2. Визуальное выделение просроченных задач

- [ ] Просроченные задачи (дедлайн < текущей даты, не завершены) выделены красным
- [ ] Завершённые задачи выделены зелёным
- [ ] Задачи в работе выделены жёлтым

**Шаги:**
1. Открыть уровень 2
2. Найти просроченную задачу
3. Проверить красную границу слева
4. Найти завершённую задачу
5. Проверить зелёную границу слева

---

#### 2.3. Обработка отсутствующих данных

- [ ] Задача без дедлайна отображается корректно
- [ ] Задача без даты начала отображается корректно
- [ ] Задача без названия отображается как "Без названия"
- [ ] Некорректные данные не вызывают ошибок

**Шаги:**
1. Открыть уровень 2
2. Проверить задачи с отсутствующими полями
3. Убедиться, что нет ошибок в консоли

---

### 3. Тестирование ошибок

#### 3.1. Обработка ошибок сети

- [ ] При ошибке сети отображается понятное сообщение
- [ ] Кнопка "Повторить" работает
- [ ] Ошибка логируется в консоль

**Шаги:**
1. Отключить интернет
2. Открыть уровень 2
3. Проверить сообщение об ошибке
4. Включить интернет
5. Кликнуть "Повторить"
6. Проверить успешную загрузку

---

#### 3.2. Обработка ошибок API

- [ ] При ошибке API отображается понятное сообщение
- [ ] Тип ошибки определяется корректно (сеть, таймаут, доступ)
- [ ] Кнопка "Повторить" работает

**Шаги:**
1. Симулировать ошибку API (404, 500)
2. Проверить сообщение об ошибке
3. Проверить работу кнопки "Повторить"

---

#### 3.3. Пустое состояние

- [ ] При отсутствии задач отображается сообщение "Нет задач для отображения"
- [ ] Иконка отображается корректно
- [ ] Нет ошибок в консоли

**Шаги:**
1. Открыть попап для сотрудника без задач
2. Перейти на уровень 2
3. Проверить отображение пустого состояния

**Шаги:**
1. Открыть попап на мобильном устройстве
2. Проверить отображение карточек
3. Проверить адаптивность пагинации
4. Проверить читаемость текста

#### 4.2. Анимации и переходы

- [ ] Анимация появления карточек работает
- [ ] Индикатор загрузки отображается корректно
- [ ] Нет "дёрганий" при загрузке

**Шаги:**
1. Переключиться между уровнями

#### 4.3. Соответствие стилям

- [ ] Стили карточек соответствуют другим попапам
- [ ] Цветовая схема единообразна
- [ ] Шрифты соответствуют проекту
- [ ] Отступы соблюдаются

**Шаги:**

### 5. Тестирование производительности

#### 5.1. Загрузка задач

- [ ] Загрузка задач не блокирует интерфейс
- [ ] Индикатор загрузки отображается во время загрузки
- [ ] Время загрузки приемлемо (< 2 секунд для 10 задач)

**Шаги:**
1. Открыть уровень 2
2. Засечь время загрузки
3. Проверить отображение индикатора

---

#### 5.2. Большое количество задач

- [ ] При большом количестве задач (50+) пагинация работает корректно
- [ ] Нет задержек при переключении страниц
- [ ] Нет утечек памяти

**Шаги:**
1. Открыть уровень 2 с >50 задачами
2. Переключаться между страницами
3. Проверить производительность (DevTools Performance)

---

#### 5.3. Переключение уровней

- [ ] Нет утечек памяти при переключении уровней
- [ ] Состояние сбрасывается корректно
- [ ] Нет накопления данных

**Шаги:**
1. Переключаться между уровнями несколько раз
2. Проверить память в DevTools
3. Убедиться, что нет утечек
