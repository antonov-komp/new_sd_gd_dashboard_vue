## Цель этапа

Спроектировать улучшенный UX/UI для секции фильтров и блока стадий: структурировать фильтры, дать понятный контейнер и состояния для стадий, спроектировать дропдаун «Сотрудник» с поиском и состояниями загрузки/ошибки/пусто.

## Контекст

- Этап 28-01 зафиксировал текущее состояние: «висящие» кнопки стадий, неструктурированные фильтры, пустое поле «Сотрудник».  
- Нужно подготовить дизайн-решение (структура, отступы, состояния, тексты, плейсхолдеры) для реализации на Vue с сохранением стиля Bitrix24 (сетка 8px, понятные бордеры, контрасты, hover/active).

## Область работ этапа

1) Спроектировать layout секции фильтров: заголовок, подзаголовок (опц.), сетка 8px, единые отступы, подписи и placeholder’ы.  
2) Спроектировать блок стадий: контейнер с фоном/бордером/радиусом, грид/флекс выравнивание, минимальная высота, состояния hover/active/disabled.  
3) Спроектировать UX выпадающего «Сотрудник»: поиск с дебаунсом, loading/empty/error, aria-label, автоподсказка.  
4) Определить текстовые состояния: placeholders, tooltips (если нужны), подсказки при пустом списке, retry при ошибке.  
5) Подготовить схему пропсов/событий для будущих компонентов (`FiltersPanel`, `StageChips`, `EmployeeSelect`), чтобы передать разработке в реализации.

## Шаги выполнения

1. Описать целевую структуру фильтров (блоки/строки/колонки), указать отступы, размер шрифта, выравнивание.  
2. Описать визуал и поведение блока стадий: фон (`#f8f9fa`), бордер (`#e5e7eb`), радиус 6–8px, паддинги 8–12px, состояния hover/active (контраст, смена бордера/фона).  
3. Описать UX для `EmployeeSelect`:  
   - Placeholder «Выберите сотрудника сектора 1С»;  
   - Loading state (спиннер/текст);  
   - Empty state («Нет сотрудников сектора 1С»);  
   - Error state (кнопка «Повторить», краткий текст ошибки);  
   - Поиск с дебаунсом (200–300 мс), ограничение выдачи (например, 20).  
4. Согласовать тип выбора стадий: single или multi-select; зафиксировать визуал для выбранного варианта.  
5. Подготовить описание пропсов/событий компонентов (черновик API):  
   - `StageChips`: `modelValue`, `options`, `multiple`, `disabled`, события `update:modelValue`, `onHover`.  
   - `EmployeeSelect`: `modelValue`, `loadOptions(search)`, `placeholder`, `disabled`, события `update:modelValue`, `onSearch`, `onRetry`.  
   - `FiltersPanel`: слоты/пропсы для конфигурации полей, кол-во колонок/брейкпоинты.  
6. Зафиксировать решения в этом файле: текстовое описание и (опц.) ссылки на макет/скрин, если доступен.
