## Цель этапа

Зафиксировать текущее состояние секции фильтров и блока стадий графика сектора 1С (DOM, стили, данные), чтобы спроектировать улучшенный UX на следующих этапах.

## Контекст

- В текущем UI кнопки стадий визуально «висят в воздухе», отсутствует явный контейнер/фон/бордер.  
- Секция фильтров не структурирована; поле «Сотрудник» пустое, без списка фактических сотрудников сектора 1С.  
- Приложение на Vue должно соответствовать Bitrix24 UI (сетка 8px, контрасты, hover/active).  

## Область работ этапа

1) Сбор артефактов текущего UI: скриншоты, DOM-структура, применённые классы/стили.  
2) Инвентаризация компонентов/файлов, отвечающих за фильтры и стадии.  
3) Анализ источников данных для поля «Сотрудник» (REST Bitrix24 или внутренний API): какой URL, формат ответа, фильтр по сектору 1С.  
4) Оценка поведения фильтров и стадий: текущие состояния (hover/active/disabled), отступы, выравнивание.  

## Шаги выполнения

1. Сделать скриншоты и текстовое описание текущего вида секции фильтров и блока стадий (в разных ширинах, если применимо).  
2. Зафиксировать DOM/классы: выписать ключевые контейнеры, flex/grid, отступы, фоны/бордеры; указать файлы, где это описано (Vue-компоненты, SCSS/CSS).  
3. Найти и перечислить компоненты/файлы:  
   - Компонент фильтров (`FiltersPanel.vue` или аналог),  
   - Компонент стадий (`StageChips.vue` или аналог),  
   - Стили, связанные с ними.  
4. Проверить поле «Сотрудник»:  
   - Какой API сейчас используется (или отсутствует),  
   - Какие параметры ожидаются (поиск/лимит/фильтр сектора),  
   - Как обрабатывается загрузка/ошибки/пустой результат.  
5. Сформировать краткий отчёт (в этом файле или отдельным блоком) с выводами и рисками для UX/реализации следующих этапов.
