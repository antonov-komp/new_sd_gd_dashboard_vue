# TASK-009: Кнопки навигации "НАЗАД" для дашборда сектора 1С

**Дата создания:** 2025-12-06 13:26 (UTC+3, Брест)  
**Статус:** Новая  
**Приоритет:** Средний  
**Исполнитель:** Bitrix24 Программист (Vue.js)

---

## Описание

Добавить кнопки навигации "НАЗАД" для дашборда сектора 1С, обеспечивающие возврат на стартовую страницу приложения. Реализация включает две отдельные задачи для разных версий интерфейса:

1. **Кнопка в заголовке** — для десктопной и планшетной версий
2. **Плавающая кнопка** — для мобильной версии

---

## Контекст

Дашборд сектора 1С является отдельной страницей в приложении, доступной по маршруту `/dashboard/sector-1c`. Пользователи переходят на дашборд со стартовой страницы (`IndexPage.vue`, маршрут `/`), но в текущей реализации отсутствует удобная возможность вернуться обратно без использования кнопки "Назад" браузера.

**Проблема:**
- На десктопе и планшете пользователям нужно использовать кнопку "Назад" браузера
- На мобильной версии при прокрутке страницы доступ к заголовку теряется
- Отсутствует явная визуальная подсказка о возможности вернуться на главную

**Решение:**
- Добавить кнопку "НАЗАД" в заголовок для десктопа и планшета
- Добавить плавающую кнопку "НАЗАД" для мобильной версии, закреплённую на уровне скролла

---

## Структура задачи

Задача разделена на две подзадачи:

### TASK-009-01: Кнопка "НАЗАД" в заголовке
**Для:** Десктоп (> 1024px) и Планшет (768px - 1024px)  
**Расположение:** В заголовке дашборда, возле текста "Дашборд - Сектор 1С"  
**Файл:** `DOCS/TASKS/TASK-009-01-back-button-header-dashboard-sector-1c.md`

### TASK-009-02: Плавающая кнопка "НАЗАД" для мобильной версии
**Для:** Мобильная версия (< 768px)  
**Расположение:** Плавающая кнопка, закреплённая на уровне скролла (fixed position)  
**Файл:** `DOCS/TASKS/TASK-009-02-floating-back-button-mobile-dashboard-sector-1c.md`

## Технические требования

### Навигация:
- Обе кнопки должны выполнять переход на стартовую страницу (`/`)
- Использовать `router.push('/')` для программной навигации
- Кнопки должны работать во всех состояниях компонента (загрузка, ошибка, контент)

### Дизайн:
- Кнопки должны соответствовать гайдлайнам Bitrix24 UI
- Использовать акцентные цвета Bitrix24
- Обеспечить визуальную обратную связь (hover, active)

### Адаптивность:
- Кнопка в заголовке: видна на десктопе и планшете, скрыта на мобильной версии
- Плавающая кнопка: видна только на мобильной версии, скрыта на планшете и десктопе

### Accessibility:
- Добавить `aria-label` для screen readers
- Обеспечить достаточный контраст для читаемости
- Кнопки должны быть доступны для навигации с клавиатуры
